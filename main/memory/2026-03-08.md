## Session 3: Architecture Deep Dive & Directory Restructuring

### Key Architecture Findings:
- Sub-agents do NOT auto-load parent workspace bootstrap files — strictly isolated
- OpenClaw does NOT walk up directory tree for bootstrap files
- Sub-agent sessions (via sessions_spawn) only inject AGENTS.md + TOOLS.md
- Independent agents (configured in openclaw.json) load full bootstrap from their own workspace
- `bootstrap-extra-files` hook is the ONLY way to auto-inject external files
- Workspace is default cwd, not hard sandbox — agents CAN read other dirs via absolute/relative paths at runtime
- Saboo's multi-agent pattern works because agents READ shared files at runtime, not via auto-bootstrap

### Saboo Article Key Takeaways:
- Filesystem = coordination layer (no API/message queues needed)
- One-writer-many-readers avoids conflicts
- SOUL.md should be 40-60 lines; user info can be embedded (no separate USER.md needed)
- Two-layer memory: daily logs + curated MEMORY.md
- Start with 1 agent, add incrementally
- Iterative refinement: first version is always mediocre

### Shared File Injection — Final Resolution:
- `bootstrap-extra-files` hook: **cannot inject files outside workspace** (realpath-checked)
- Symlinks in workspace pointing to external files: **also blocked** (shows `[MISSING]`)
- GitHub Issue #21802 confirms this is a known limitation — feature requested but not implemented
- Symlink blocking is intentional: CVE/GHSA-fgvx-58p6-gjwc (symlink traversal vulnerability)
- Saboo's subdirectory pattern does NOT solve this either — parent files still outside sub-workspace
- **Conclusion**: No way to auto-inject shared bootstrap files across agents. Only option: copies or runtime `read`
